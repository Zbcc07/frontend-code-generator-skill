# 前端代码生成规范

此 Skill 定义了基于 Vue3 + TypeScript + SCSS 的前端项目开发规范，确保代码质量、可维护性和一致性。

## 触发条件

当用户请求以下任务时，使用此 Skill：
- 生成前端页面或组件代码
- 基于 Figma 设计稿开发页面
- 创建 Vue3 + TS + SCSS 项目
- 开发需要数据可配置性的页面

## 技术栈规范

### 默认技术栈
- **框架**: Vue 3 (Composition API)
- **语言**: TypeScript
- **样式**: SCSS
- **开发模式**: 组件化开发

### 命名规范
- 变量名、class 名遵循**简单易读**原则
- 使用语义化命名，避免缩写（除非是通用缩写如 `btn`、`img`）
- 组件名使用 PascalCase（如 `UserProfile.vue`）
- 变量/函数名使用 camelCase
- CSS class 使用 kebab-case

### 可配置性原则
- 所有页面组件需支持**可配置化**
- 配置与视图分离，便于后续维护和接口对接

## 文件组织规范

### 目录结构

```
project/
├── public/                    # 公共资源
│   ├── images/               # 图片资源
│   │   ├── home/            # 首页相关图片
│   │   ├── about/           # 关于页相关图片
│   │   └── shared/          # 通用图片
│   └── css/                 # 全局样式
│       └── global.scss
├── src/
│   ├── views/               # 页面视图
│   │   └── [功能模块]/      # 按功能模块创建文件夹
│   │       ├── index.vue    # 页面主文件
│   │       ├── components/  # 页面级组件
│   │       └── css/         # 页面样式统一存放
│   │           └── index.scss
│   ├── components/          # 全局公共组件
│   ├── api/                 # 接口定义
│   ├── mock/                # Mock 数据
│   ├── data/                # 本地 JSON 数据源
│   └── types/               # TypeScript 类型定义
```

### 文件存放规则
1. **图片资源**: 统一放在 `public/images/` 下，按功能模块创建子文件夹
2. **组件样式**: 所有组件样式必须单独抽出，分散到各组件文件中管理
3. **组件**: 页面级组件放在 `views/[模块]/components/`，全局组件放在 `src/components/`

## Figma 设计稿还原规范

### 开发流程
1. **先规划布局**: 严格按照设计稿还原前，先规划整体布局结构，与用户确认布局合理性
2. **确认后再开发**: 布局确认后再进行详细开发

### 背景实现规范
- **优先使用 `background` 属性**实现设计稿背景
- **避免使用 `<img>` 标签**进行背景定位
- 背景图片放在 `public/images/` 对应模块目录下

### 响应式布局规则

根据**设计稿类型**决定是否采用响应式布局：

| 设计稿类型 | 响应式要求 | 说明 |
|-----------|-----------|------|
| **PC 设计稿** | **必须响应式** | 适配移动端，采用 CSS Media Query |
| **移动端设计稿** | **可选** | 默认不强制适配 PC，如需适配需明确说明 |

**断点建议：**
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

**开发前确认：**
- [ ] 确认设计稿类型（PC / 移动端 / 双端）
- [ ] PC 设计稿：默认响应式适配移动端
- [ ] 移动端设计稿：询问是否需要适配 PC

## 数据可配置性规范

### 数据渲染方式
每个页面需支持**两种数据渲染方式**：

1. **本地 JSON 数据**: 从 `src/data/` 目录下的 JSON 文件获取数据
2. **Mock 接口**: 模拟真实接口获取数据，便于后续接口对接调试

**用户可通过配置选择使用 JSON 或 Mock 的数据源**，通过修改配置文件或环境变量切换数据源，无需修改业务代码。

### 文件要求
每个页面必须包含：
1. **接口文档**: 参考 `references/api-doc-template.md` 格式生成
2. **本地 JSON 数据源**: `src/data/[page-name].json`
3. **Mock 模拟接口**: `src/mock/[page-name].ts`

### 生成顺序（重要）

**必须按以下顺序生成，用户确认后再进行下一步：**

```
1. 接口文档（先） →  用户确认入参和接口拆分  →  2. Mock 接口 + JSON 数据（后）
```

#### 第一步：生成接口文档
- **一个页面对应一个接口文档**（如 `docs/home-api.md`）
- 文档内需要拆分几个接口，**需与用户确认**
- **必须让用户确认接口入参**（请求参数、返回值结构）
- 确认通过后再进行下一步

#### 第二步：生成 Mock 接口和 JSON 数据
- **Mock 接口必须根据已确认的接口文档实现**
- **Mock 数据结构必须与文档保持一致**
- JSON 数据作为本地数据源，结构与 Mock 返回数据一致

### 接口文档格式
接口文档需包含：
- Controller 名称
- API 基础路径
- 统一响应格式说明
- 接口列表（地址、方法、参数、返回值）
- 字段说明
- 数据表结构设计
- Mock 数据说明
- 后端对接指南

## 开发流程与检查清单

### 标准开发流程

```
1. 分析需求/设计稿
   ↓
2. 【用户确认】规划整体布局
   ↓
3. 生成接口文档（一个页面一个文档）
   ↓
4. 【用户确认】接口入参、接口拆分
   ↓
5. 生成 Mock 接口 + JSON 数据（根据已确认文档）
   ↓
6. 开发页面组件
   ↓
7. 完成
```

### 开发检查清单

生成代码前，确认以下事项：
- [ ] 是否使用 Vue3 + TypeScript + SCSS
- [ ] 组件是否具备可配置性
- [ ] 文件目录是否符合规范
- [ ] 如果是 Figma 设计稿：是否先确认布局
- [ ] 背景是否优先使用 background 实现
- [ ] 是否确认设计稿类型（PC / 移动端）
- [ ] PC 设计稿是否实现响应式适配移动端
- [ ] 移动端设计稿是否需要适配 PC（如需要）

**数据相关（必须按顺序）：**
- [ ] **先生成接口文档**（一个页面一个文档）
- [ ] **用户确认接口入参和接口拆分**
- [ ] **再根据文档生成** Mock 接口和 JSON 数据
- [ ] Mock 数据结构是否与文档一致
